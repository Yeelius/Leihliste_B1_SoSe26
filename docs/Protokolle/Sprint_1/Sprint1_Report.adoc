== 6. Anpassung der Arbeitsweise für Sprint 2

Beschreiben Sie, was Ihr Team im nächsten Sprint beibehalten, ändern oder gezielt verbessern will. Leitfragen:

* Welche Arbeitsweise hat gut funktioniert?
* Was hat Ihre Zusammenarbeit behindert?
* Was wollen Sie in Sprint 2 konkret anders machen?
* Welche offene Frage soll Sprint 2 beantworten?

[cols="1,3", options="header"]
|===
| Kriterium | Ihre Antwort

| Beibehalten
| Regelmäßigkeit: Die Meetings fanden regelmäßig statt und die Agenda wurde eingehalten.

Teamkultur: Alle waren anwesend, die Kommunikation war offen, freundlich und wertschätzend.

Qualität: Es besteht ein gemeinsamer Qualitätsanspruch an die Arbeitsergebnisse.

Vision: Das Team arbeitet gemeinsam an der Produktvision.

| Ändern / verbessern
| Zeitmanagement: Aufgaben wurden oft auf den letzten Drücker erledigt; Meetings liefen über die angesetzte Zeit (Timeboxen wurden nicht eingehalten). -> Bessere individuelle Vorbereitung auf Meetings, um Entscheidungszeit zu verkürzen. Strengere Einhaltung von Timeboxen (ggf. Themen auslagern). Agenda wird auf Github gepflegt (alle können Punkte hinzufügen).

Transparenz: Es fehlte der Überblick über anstehende Aufgaben im Sprint. -> Alle Aufgaben werden im Sprint Planning erfasst (inkl. Wochenplanung). Github Issues werden konsequent als Tasks genutzt (bei gemeinsamen Tasks Sub-Issues erstellen). Deadlines werden direkt in Github Issues gesetzt.

Kommunikation: Zu wenig Updates zum Fortschritt der Aufgaben; gleichzeitig führte eine Flut an Nachrichten im Discord zu Informationsüberlastung (ohne relevante Markierungen). -> Regelmäßige Updates im Discord oder kurze Status-Meetings in der Woche. Relevante Nachrichten werden mit einem Ping (@Person) markiert.

Organisation: Die Terminfindung in Meetings nahm zu viel Zeit in Anspruch. -> Terminfindung erfolgt 2 Tage vorab via Nuudel.
